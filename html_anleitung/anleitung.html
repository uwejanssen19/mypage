Kurzanleitung Website

−Einleitung‍

Die vorliegende Kurzanleitung zum Erstellen einer Website ist als Einstieg in die HTML-Programmierung gedacht. Sie besteht aus den drei Dateien index.html (diese Datei, enthält den eigentlichen HTML-Code), anzeige.css (enthält die Formatierungs­angaben in CSS) und funktionen.js (enthält die JavaScripts für die dynamischen Elemente).
Das Speichern der Lesezeichen funktioniert nur mit aktiviertem JavaScript und aktivierten Cookies, diese sind dann auch nur auf dem gleichen Rechner gespeichert. Das Kapitel PHP ist derzeit noch im Aufbau.

? Eine Website besteht aus mehreren Seiten, die miteinander verlinkt sind. Die Seiten sind in HTML geschrieben. Die hierarchisch angeordneten Elemente dieser Auszeichnungs­sprache bezeichnet man als Tags.
Als Beispiel erstellen wir eine vierseitige Website in zwei Sprachen. Von jeder Seite aus soll es möglich sein, die anderen Seiten der gleichen Sprache zu erreichen, sowie die anders­sprachigen Seiten gleichen Inhalts.

Zum Erstellen einer Website benötigt man einen Texteditor (zu empfehlen ist z.B. Notepad++) und ein Malprogramm (wie z.B. Paint.net), sowie bei Bedarf Programme zum Bearbeiten von Photos, Audio oder Video. Dieses Beispiel arbeitet ohne Bilder, die grafischen Elemente werden komplett mit CSS erzeugt.
Zum Betrachten des Ergebnisses installiert man möglichst mehrere Browser (z.‍B. Mozilla Firefox, Google Chrome und Internet Explorer). Um das Ergebnis schließlich ins Netz zu stellen benötigt man ein FTP-Programm (Total Commander).

−Tipps‍

Eine Website besteht aus verschiedenen Komponenten, die getrennt voneinander bearbeitet werden sollten. Als Vorbereitung für den grafischen Aufbau kann man Skizzen erstellen, ebenso sollte man Texte, Bilder, Mediendateien sammeln, die möglicherweise für das Projekt verwendet werden. Für die Arbeit an den Seiten benötigt man außerdem Platzhalter; dafür eignen sich Bilder, die als Beispieldateien mit dem Betriebssystem geliefert wurden, sowie Blindtexte (Lorem ipsum). Hilfreich sind außerdem Farb- und Zeichentabellen, frei verfügbare Schriften und Symbole.

In diesem Verzeichnis erstellen wir ein Unterverzeichnis namens data für die verschiedenen Dateien und Icons, auf die die Webseite zugreift. Weitere Verzeichnisse wie bilder für die Bilder, audio und video für Mediendateien und news für zu aktualisierende Dateien sind natürlich möglich.

−Tags‍

HTML-Dokumente bestehen aus Tags, das sind Anweisungselemente in spitzen Klammern. Sie können Attribute und (in der Regel angezeigten) Inhalt haben. Man unterscheidet zwischen verschiedenen Arten:

inline-tag

Inline-Tags bestehen aus einem Anfangs- und einem End-Tag gleichen Namens, jeweils mit spitzen Klammern umschlossen; dem Namen des End-Tags ist ein Schrägstrich vorangestellt. Der Inhalt steht zwischen den beiden Tags. Inline-Tags erzeugen keinen Absatz.

Beispiel inline-tag

Dieses Wort wird <b>fett</b> dargestellt.

blocklevel-tag

Blocklevel-Tags bestehen aus einem Anfangs- und einem End-Tag gleichen Namens, jeweils mit spitzen Klammern umschlossen; dem Namen des End-Tags ist ein Schrägstrich vorangestellt. Der Inhalt steht zwischen den beiden Tags. Blocklevel-Tags erzeugen einen Absatz.

Beispiel blocklevel-tag

<p>Dieses ist ein Textabschnitt.</p>

stand-alone-tag

Stand-alone-Tags werden mit einer öffnenden spitzen Klammer < eingeleitet und enden mit der Zeichenfolge  /> (Leerzeichen-Schrägstrich-spitze Klammer zu) − sie haben keinen (angezeigten) Textinhalt.
Die Schreibweise mit dem Schrägstrich wird verwendet, um kompatibel mit dem XHTML-Format zu sein (in HTML würde ein einfaches > genügen).

Beispiel stand-alone-tag

Es folgt ein Bild: <img src="bild.jpg" alt="Bild">

−Browserweichen‍

Da der Internet Explorer zahlreiche HTML-Tags nicht versteht beziehungsweise falsch interpretiert, wurden mit Version 5 conditional comments eingeführt, die nur vom IE (ab Version 5) verstanden, von anderen Browsern jedoch als Kommentar interpretiert (also ignoriert) werden. Damit ist es möglich, bestimmten HTML-Code speziell für den IE anzugeben.
Für conditional comments schreibt man innerhalb eines Kommentars eine if-Bedingung in eckigen Klammern.

Beispiel HTML-Browserweiche

<!--[if lte IE 6]>
  Sie benutzen einen Internet Explorer (Version kleiner gleich 6).
<![endif]-->

Dieser Text wird nur von Internet Explorer Version 5 oder 6 angezeigt, andere Browser zeigen nichts an.

Als Bedingung nach dem if kann die Version angegeben werden: IE für alle Versionen, IE 5, IE 6 oder IE 7 für eine spezielle Version. Außerdem kann als Prefix ein Operator angegeben werden:

conditional comment prefixes
!
nicht (not)
lt
kleiner als (lower than)
lte
kleiner als oder gleich (lower than or equal)
gt
größer als (greater than)
gte
größer als oder gleich (greater than or equal)

Um die Möglichkeit zu haben, die in HTML5 neu hinzugekommenen strukturellen Tags zu benutzen, kann man im Header der HTML-Datei für den Internet Explorer (bis zur Version 8) auf die folgende JavaScript-Datei verweisen:

HTML5 script link
<!--[if lt IE 9]>
    <script type="text/javascript"
            src="http://html5shiv.googlecode.com/svn/trunk/html5.js">
    </script>
<![endif]-->

Das funktioniert allerdings nur, wenn der Benutzer JavaScript nicht deaktiviert hat.

−& Sonderzeichen‍

? Der Text in einem HTML-Dokument (in unserem Basis-Dokument-Beispiel braun dargestellt) wird mit einigen Ausnahmen so angezeigt, wie eingegeben. Steuerzeichen (Absätze, Tabulatoren) und Leerzeichen werden grundsätzlich zu einem Leerzeichen zusammengefasst, werden also nicht dargestellt. Wird im head der Datei der Zeichensatz UTF-8 angegeben, werden alle direkt eingegebenen Sonderzeichen korrekt angezeigt (mit Ausnahme der Zeichen < und &).

Über die Tastatur nicht erreichbare Sonderzeichen (nicht-lateinische Zeichen, Anführungszeichen) können über den Unicode (zwischen & und ;) eingegeben werden, einige haben standardisierte Namen (named entities). Der Unicode kann dezimal nach einem einleitenden # oder hexadezimal nach einem #x angegeben werden.

Der Internet Explorer versteht einige named entities nicht (z.B. &apos; und mehrere mathematische und grafische Symbole), deshalb wird in der folgenden Auflistung in diesen Fällen der Unicode angegeben.

Da einige Zeichen in HTML eine bestimmte Bedeutung haben, müssen diese – wenn sie angezeigt werden sollen – grundsätzlich kodiert werden (das Zeichen < leitet einen Tag ein, das Zeichen & eine Entity). Außerdem sollte das @ nicht unkodiert auf der Seite auftauchen, damit Email-Scanner die Adressen nicht so einfach finden.

−Vererbung‍

Bestimmte Eigenschaften (z.B. Textattribute) werden vom Elternelement vererbt, andere (z.B. Ränder) nicht. Möchte man diese Eigenschaften trotzdem vererben, notiert man im Kindelement das Schlüsselwort inherit.

Beispiel Vererbung
div { border:1px solid black; }
p { border:inherit; }

Im Beispiel bekommt das p-Tag einen schwarzen Rand, wenn es innerhalb des schwarz umrandeten div-Tags auftaucht.

Eigenschaften werden von spezifischeren Zuweisungen überschrieben. Das lässt sich mit dem Schlüsselwort !important blockieren.

Beispiel feste Eigenschaft
var { background-color:blue; color:black !important; }
.special { color:blue; font-weight:bold; }

Im Beispiel werden Variablen blau hinterlegt, Textfarbe ist schwarz. Elemente mit der Klasse special werden blau und fett geschrieben, var-Tags mit der Klasse special werden aber in der Schriftfarbe schwarz geschrieben - das Schlüsselwort !important erzwingt die schwarze Schrift für alle var-Tags.

Microsoft-Filter

Für den Internet-Explorer wurden spezielle Filter entwickelt, mit denen man einige der nicht interpretierten CSS-Eigenschaften mit dem IE (ab Version 5.5) darstellen kann. Sie werden von modernen Browsern ignoriert.

Beispiel IE Filter
.shadowedbox {
    zoom:1;
    filter:progid:DXImageTransform.Microsoft.DropShadow
	(color=#000,offx=1mm,offy=2mm);
    ms-filter:progid:DXImageTransform.Microsoft.DropShadow
	(color=#000,offx=1mm,offy=2mm);
    box-shadow:1mm 2mm #000;
}

Die Microsoft-Filter funktionieren nur mit Elementen, die die (Microsoft-proprietäre) Eigenschaft hasLayout haben. Das sind einige Blockelemente (body, table, object, input, image und hr), sowie Elemente, bei denen bestimmte Attribute gesetzt wurden (width, height, float, position:absolute, display:inline-block sowie die MS-proprietäre Eigenschaft zoom). Um also sicher zu gehen, dass der Filter auch angezeigt wird, kann man das Attribut zoom:1; setzen. Es wird von modernen Browsern ignoriert.

? Farben werden durch die Angabe des Rot-, Grün- und Blauanteils definiert. Möglich ist eine (drei mal zweistellige) Hexadezimalangabe oder die Angabe der Einzelwerte als Dezimalwert (bis 255) oder in Prozent; eine Hexadezimalangabe mit doppelten Ziffern kann (drei mal einstellig) abgekürzt werden.

Der default-Wert für den Hintergrund eines Bereiches ist transparent.

blue = #0000FF = #00F = rgb(0,0,255) = rgb(0%,0%,100%) = hsl(240,100%,50%)

Websichere Schriften xxx

Mit der Eigenschaft font-family kann man einem Element eine bestimmte Schrift zuweisen. Für den Fall, dass diese auf dem Computer des Benutzers nicht verfügbar ist, lassen sich mehrere spezifische Alternativschriften (mit Komma getrennt) angeben. Grundsätzlich sollte diese Liste mit einer generischen Schriftfamilien-Angabe (serif, sans-serif, monotype, cursive) enden, dann wird zumindest eine verwandte Schrift zur Darstellung verwendet.

Spezielle Schriften laden

Es ist mit CSS3 möglich, einen speziellen Schriftfont zu laden, der dann als font-family benutzt werden kann.

Beispiel Schrift laden
CSS3 
@font-face { font-family: myFont;
    src: url('myFont-Roman.otf'); }
@font-face { font-family: myFont;
    font-weight: bold; src: url('myFont-Bold.otf'); }

Der Schriftfont kann dann unter dem Namen 'myFont' als font-family benutzt werden.

−Dynamic (JavaScript)‍

JavaScript ist eine Programmiersprache als Ergänzung zu HTML, die zur Laufzeit vom Browser ausgeführt wird. Mit JavaScript können Bildwechsel- und andere Effekte erzeugt werden.

−Implementierung‍

−JavaScript-Bereiche definieren‍

JavaScript-Anweisungen können in einer separaten Datei oder in der HTML-Datei stehen; meist schreibt man die Anweisungen dann in den Header. Einfache Funktionen können auch direkt im Tag angegeben werden.

Im folgenden Beispiel wird beim Anklicken des div-Tags und beim Anklicken des a-Tags ein Informationsfenster angezeigt.

a) Anweisung direkt im HTML-Tag:
Beispiel HTML-Datei
<!DOCTYPE html>
<html lang="de"><head>
	...
	<meta http-equiv="Content-Script-Type" content="text/javascript">
	...
</head><body>
	...
	<div onclick="alert('Hallo');">Meldung anzeigen</div>
	...
	<a href="javascript:alert('Hallo');">Meldung anzeigen</a>
	...
</body></html>

Da die von den beiden Tags aufgerufene Funktion nur aus einer Anweisung besteht (nämlich der Anzeige des alert-Fensters), kann sie - wie in diesem Beispiel gezeigt - direkt in den Tag geschrieben werden. In diesem Fall muss der übergebene Text in einfache statt doppelte Hochkommata gesetzt werden (die doppelten Hochkommata kennzeichnen ja die gesamte Attributeigenschaft).

Während der Eventhandler onclick bei allen Tags benutzt werden kann, gibt es für den a-Tag auch die Möglichkeit, mit dem Attribut href auf ein JavaScript zu referenzieren; dann muss als Referenzziel javascript: vor der Funktion angegeben werden.

b) Anweisungen im Header (oder Body) der HTML-Datei:
Beispiel HTML-Datei
<!DOCTYPE html>
<html lang="de">
	...
	<script type="text/javascript"> /* <![CDATA[ */
	    function meldung() { alert("Hallo"); }
	/* ]]> */ </script>
	...
	<div onclick="meldung();">Meldung anzeigen</div>
	...
</html>

Der Eventhandler onclick ruft im oberen Beispiel die Funktion meldung() auf. Diese steht an anderer Stelle in einem script-Tag und besteht aus der Anzeige des alert-Fensters.

Beispiel HTML-Datei
<!DOCTYPE html>
<html lang="de">
	...
	<script type="text/javascript"> /* <![CDATA[ */

    // Methoden
	    function meldung() { alert("Hallo"); }
    // Event-Überwachung
	    window.onload = function() {
		document.getElementById("klickbox").onclick = meldung;
	    }

	/* ]]> */ </script>
	...
	<div id="klickbox">Meldung anzeigen</div>
	...
</html>

In diesem Beispiel erhält das div-Tag nur eine id zur Identifizierung. Die Event-Überwachung sowie die Methode stehen im script-Bereich. Beim Laden des Dokuments wird das Element mit der id klickbox überwacht, und bei Aktivierung durch einen Klick wird dem Element die Funktion meldung als Objekt (deshalb fehlen hier die Klammern) übergeben, mit anderen Worten, die Funktion wird ausgeführt.

c) Anweisungen in einer separaten JavaScript-Datei:
Beispiel HTML-Datei
<!DOCTYPE html>
<html lang="de"><head>
	...
	<script type="text/javascript" src="funktionen.js"></script>
	...
</head><body>
	...
	<div id="klickbox">Meldung anzeigen</div>
	...
</body></html>
In der JavaScript-Datei funktionen.js steht:
// Methoden
    function meldung() { alert("Hallo"); }
// Event-Überwachung
    window.onload = function() {
	document.getElementById("klickbox").onclick = meldung;
    }

Am übersichtlichsten ist es, den kompletten JavaScript-Code (ebenso wie die CSS-Formatierungen) in eine eigene Datei zu schreiben und im HTML-Dokument darauf zu verweisen. Dafür benötigt man im Allgemeinen zwei Ereignisse: Das Ereignis onload des windows-Objektes wartet, bis die Seite vollständig geladen ist, die Ereignisse onclick der einzelnen Elemente überwachen die Mausklicks.

Soll nur eine Funktion ohne Parameterübergabe aufgerufen werden (im Beispiel die Funktion meldung), wird diese als Objekt ohne runde Klammern dem Handler zugewiesen: 

Element.Eventhandler = Funktionsname;

Sollen mehrere Funktionen aufgerufen oder Parameter übergeben werden (im Beispiel der onload-Aufruf), verwendet man eine namenlose Funktion: 

Element.Eventhandler = function() {
 Funktionsname (Parameter);
}

−Bezeichner‍

JavaScript ist eine dynamisch typisierte Sprache, d. h., der Typ einer Variable kann sich während der Ausführung ändern; die Zuordnung zu einem Typ geschieht bei der Zuweisung eines Wertes an die Variable und kann sich bei jeder Operation ändern. Um eine Variable zu benutzen, wird sie zunächst deklariert, dann wird ihr ein Wert zugewiesen. Die gleichzeitige Deklaration und Wertzuweisung heißt Initialisierung.

−Variablendeklaration‍

Variablen für primitive Objekte repräsentieren das Objekt selber. Das bedeutet, wenn man eine Variable ändert, nachdem man sie kopiert hat, stehen in beiden Variablen unterschiedliche Werte – man hat zwei Variablen.

−Objektinstanziierung‍

Objekte im engeren Sinne (also komplexe oder strukturierte Objekte) sind Daten des Referenz-Typs object, die Eigenschaften (Attribute) und Methoden (Funktionen) haben können. Methoden werden von runden Klammern gefolgt, in denen die benötigten Parameter stehen (das Klammerpaar kann auch leer sein), Eigenschaften haben keine runden Klammern.

Um ein Objekt benutzen zu können, muss man i.d.R. zunächst eine Instanz des Objektes erzeugen (mit dem Schlüsselwort new und von einem runden Klammerpaar gefolgt), danach kann man auf die Eigenschaften und Methoden des Objektes zugreifen (diese werden durch einem Punkt von der Variablen getrennt). Die Erstellung eigener Objekte mit Methoden wird im Kapitel Objekte erstellen beschrieben.

Beispiel JavaScript Objekt (klassische Instanziierung)
var Buch = new Object(); /* Erzeugung der Instanz */
Buch.Titel = "Das Buch"; /* Initialisierung der Attribute */
Buch.Autor = "P. Meyer";

Alternativ zur Erzeugung der Instanz mit dem Schlüsselwort new kann man ein solches allgemeines Objekt als Literal instanziieren. Dafür werden geschweifte Klammern verwendet, die nicht mit einem Semikolon abgeschlossen werden müssen.

Beispiel JavaScript Objekt (Instanziierung als Literal)
var Buch = {} /* Erzeugung der Instanz */
Buch.Titel = "Das Buch"; /* Initialisierung der Attribute */
Buch.Autor = "P. Meyer";

/* Kurzschreibweise: */
var Buch = {Titel:"Das Buch", Autor:"P. Meyer"}

Um ein abgeleitetes Objekt (im folgenden Beispiel das vordefinierte Objekt Date) benutzen zu können, muss man die Instanziierung mit dem Schlüsselwort new vornehmen. Das Objekt Date hat ausschließlich Methoden, keine Attribute.

Beispiel JavaScript Date-Objekt
var Datum = new Date(); /* Erzeugung der Instanz */
alert(Datum.toLocaleString()); /* Anzeige Datum */
alert(Datum.getDate()); /* Anzeige Monatstag */

var Stichtag = new Date(2010,9,25); /* = 25.10.2010 */
Stichtag.setDate(15); /* = 15.10.2010 */

Beide Instanziierungen im Beispiel initialisieren automatisch das Objekt, im ersten Fall (wo keine Parameter übergeben werden) steht die aktuelle Zeit des Benutzer-Computers in der Variablen - das ist eine Besonderheit des Date-Objekts. Im zweiten Fall wird der Variablen ein Datum zugewiesen. Monate werden von 0 gezählt, also 9 = Oktober.

Variablen für komplexe Objekte zeigen auf das Objekt. Das bedeutet, wenn man über eine Variable eine Eigenschaft des Objektes ändert, nachdem man die Zeigervariable kopiert hat (ohne für die zweite Variable eine neue Instanz zu erzeugen), verweisen beide Variablen auf das geänderte Objekt – man hat ein Objekt mit zwei Zeigervariablen.

Beispiel JavaScript copying structured objects
var A = {Titel:"Das Buch", Autor:"P. Meyer"}
var B = {}
B = A;
B.Autor = "P. Schultze";
/* A hat den Autoren-Wert "P. Meyer", B hat den Autoren-Wert "P.Schultze" */
var C = A;
C.Autor = "P. Schmidt";
/* sowohl A als auch C zeigen auf das Objekt mit "P. Schmidt" als Autor */

Der URL der aktuellen Seite kann mit document.URL oder mit location.href (ohne Klammern) ausgelesen werden. URLs sind eine besondere Form des URI, der auch für Bücher und andere Medien verwendet wird und nicht unbedingt eine Internet-Adresse darstellen muss.

Mit dem Objekt location hat man Zugriff auf die Teile des URL, wobei hostname, pathname und search noch weiter unterteilt werden können.

Der hostname besteht aus einer Kette der Domains, getrennt durch Punkte; die letzte Domain bezeichnet man als Top Level Domain, die anderen als Subdomains.

Der pathname enthält den vollständigen Pfad mit der Datei. Die Verzeichnisse werden mit einem Slash abgeschlossen.

Den Querystring mit den Variable/Wert-Paaren, die von einem Formular mit der GET-Methode übergeben wurden, kann man mit search auslesen. Die einzelnen Paare sind durch Ampersands getrennt, zwischen Bezeichner und Wert ist ein Gleichheitszeichen.

    var teil = decodeURI(location.hostname); // hostname dekodieren
    var p = teil.lastIndexOf("."); // Zeiger auf letzten Punkt setzen
    this.domains = teil.split("."); // ein Array mit den Domains
    this.TLD = teil.substr(p+1); // Toplevel-Domain

    p = teil.lastIndexOf("/"); // Zeiger auf letzten Slash setzen
    var pfad = teil.substr(0,p);
    var datei = teil.substr(p+1);
    p = datei.indexOf("."); // Zeiger auf ersten Punkt im Dateinamen setzen
    this.path = pfad; // Pfadstring (ohne Datei)
    this.folders = pfad.split("/"); // Pfadliste (als Array)
    this.filename = datei; // Dateiname (mit Extension)

−Rekursive Funktionen‍

Rekursive Funktionen rufen sich selbst auf. Da die als lokal deklarierten Variablen (dazu gehören auch die Übergabevariablen im Funktionskopf) nicht von außen gesehen werden, können sie auch die gleichen Namen haben, ohne dass sie überschrieben werden.

Eine Rekursion besteht aus der Lösung des Problems für den einfachsten Fall (das ist die Abbruch­bedingung für die Rekursionsschritte) und Anweisungen, das Problem schrittweise zu vereinfachen und aus der vereinfachten Lösung auf die eigentliche Lösung zu schließen. Beispielsweise ist die Fakultät einer Zahl das Produkt der Zahl mit der Fakultät des Vorgängers, wobei 0! = 1 festgelegt wird.

Das Beispiel zeigt eine rekursive Lösung der Türme von Hanoi. Hierbei muss ein Stapel von Scheiben unterschiedlicher Größe von A nach C (über einen Hilfsstapel B) gelegt werden, ohne dass eine größere Scheibe auf einer kleineren liegt.

function verschiebe (n, A, B, C) {
// A = Start, B = Hilfsstab, C = Ziel
// verschiebt den Turm von A über B nach C
// und schreibt die Züge ins Dokument
  if (n > 0) {

    verschiebe (n - 1, A, C, B);
    document.write
     ("<br> Zug von ",A," nach ",C);
    verschiebe (n - 1, B, A, C);

  }
} // end function verschiebe

var anzahl =
  prompt("Scheibenzahl (1 bis 8) : ",3);
anzahl = (anzahl >>> 0) % 9;
document.write
  ("Lösung für ", anzahl , " Scheiben:");
verschiebe
  (anzahl, "Turm A", "Turm B", "Turm C");

Das Programm startet in einem HTML-Dokument sofort, da die Anweisungen dafür auf der untersten Ebene stehen. Die etwas seltsame Anweisung anzahl = (anzahl >>> 0) % 9; gewährleistet durch die automatische Typumwandlung, dass die Eingabe immer eine positive ganze Zahl kleiner als 9 ist.

Die Funktion verschiebe ruft sich selbst zweimal auf, wobei die Reihenfolge der Parameter vertauscht ist. In jedem Schritt wird der (Teil-)Turm ohne die unterste Scheibe auf die Hilfsposition gesetzt, die unterste Scheibe auf das Ziel, und dann der (Teil-)Turm auf des Ziel. Dabei ändern sich jedes Mal die Positionen. Das Ende der Rekursion ist erreicht, wenn der (Teil-)Turm leer ist.

−Objekte erstellen‍

JavaScript bietet die Möglichkeit, eigene Objekte zu erstellen. Dazu wird zunächst eine Konstruktor-Funktion mit den lokalen Eigenschaften (das sind die gundlegenden Attribute, die jede Instanz des Objekts erhält) für den Prototypen definiert.

prototype constructor
rectangle = function (breite, hoehe) {
    this.width = breite;
    this.height = hoehe;
}

Das Schüsselwort this bezieht sich auf das umschließende Objekt, also die Konstruktor-Funktion selbst.

Jeder Konstruktor hat das implizite Attribut prototype, über die nun weitere, vom Prototypen an die einzelnen Instanzen vererbten Eigenschaften und Methoden hinzugefügt werden können. Während für die lokalen Eigenschaften für jede Instanz Speicherplatz reserviert wird, existieren die geerbten Eigenschaften und Attribute nur im Prototypen, belegen also nur dort Speicherplatz.

prototype extensions
rectangle.prototype.area = function() {
    return this.width * this.height;
}
rectangle.prototype.perimeter = function() {
    return 2 * (this.width + this.height);
}
rectangle.prototype.diagonal = function() {
    var that = this;

    function squaresum() {
	var sqrWidth = Math.pow(that.width, 2);
	var sqrHeight = Math.pow(that.height, 2);
	return sqrWidth + sqrHeight;
    }

    return Math.sqrt(squaresum());
}


Dem Objekt rectangle werden die Attribute Fläche, Umfang und Diagonale hinzugefügt.

Die letzte Erweiterung benutzt (zur Demonstration) eine Unterfunktion; innerhalb dieser ist das this-Objekt nicht zugänglich, daher muss zuvor die Übergabe-Variable that erstellt werden.

Von dem so erstellten Prototyp rectangle können nun Objekt-Instanzen erstellt werden und man hat Zugriff auf die definierten Eigenschaften und Methoden.

public, private und privileged

Eigenschaften, die innerhalb des Konstruktors mit dem Schlüsselwort this definiert wurden, sind öffentlich (public) und können von außen angesprochen werden. Eigenschaften und Methoden, die über den Prototypen hinzugefügt wurden, sind ebenfalls öffentlich.
Methoden, die mit this definiert wurden, sind priviligiert (privileged), das heißt, sie können von außen und von den öffentlichen Methoden angesprochen werden, allerdings nicht nachträglich geändert.

public and privileged members
function Constructor(...) {
  this.membername = value;
  this.membername = function(...) { }
}
Constructor.prototype.membername = value;
Constructor.prototype.membername = function(...) { }

Eigenschaften und Methoden, die innerhalb des Konstruktors mit var definiert wurden, sind privat (private) und nicht von außen, auch nicht von den eigenen öffentlichen Methoden, ansprechbar. Sie sind nur von den eigenen privaten Methoden ansprechbar. (Die übliche Definition einer Funktion ist eine Kurzschreibweise für die Definition mit var.)

private members
function Constructor(...) {
  var membername = value;
  var membername = function membername(...) { } // ausführliche Schreibweise
  function membername(...) { } // Kurzschreibweise
}
